# Implementation Plan: Migrate Portfolio to Next.js + React + TypeScript

> Created: 2026-07-14 19:15

## Purpose / Big Picture

Rebuild the existing static HTML portfolio (Ellah Services) as a modern Next.js application with React and TypeScript — preserving every pixel of layout, content, and design. The form stays as `mailto:` for now. The result will be a clean, type-safe, component-based project ready for Docker/Dokploy deployment.

## Target Stack

| Layer | Choice | Why |
|-------|--------|-----|
| Framework | Next.js 15+ (App Router) | Industry standard, file-based routing, server components by default |
| Language | TypeScript strict | Type safety for data, props, and event handlers |
| CSS | Global CSS file | User wants to keep existing styles verbatim, no rebuild |
| Data | Single `src/data/content.ts` file | User preference — one source of truth for all site content |
| Icons | Inline SVG → React SVG (jsx) | Same visual, no extra dependency |
| Deployment | Docker (nginx:alpine) | Works with Dokploy |

## Tasks

### 1. Initialize Next.js project

Run in the project root:
```bash
npx create-next-app@latest . --typescript --app --src-dir --import-alias "@/*" --use-npm --no-tailwind --no-eslint
```

This sets up: `src/app/`, `tsconfig.json`, `package.json`, `next.config.ts`, `public/`, `.gitignore`.

Remove auto-generated boilerplate: `src/app/fonts/`, the default `page.tsx` content, and default `globals.css` content.

### 2. Move assets to `public/`

```bash
mv assets public/assets
```

Update the favicon link path from `./assets/favicon.png` to `/assets/favicon.png`.

### 3. Extract CSS

Copy all CSS from `index.html` `<style>` (lines 9–1123) into `src/app/globals.css`, replacing the auto-generated content. Keep the `:root` variables, all selectors, `@keyframes`, and `@media` queries exactly as-is.

### 4. Create `src/data/content.ts`

Extract all dynamic content into a single typed data file:

- **Nav links** — items and their hrefs
- **Hero** — tagline, title (with em spans), subtitle, button labels+links, floating stat cards
- **About** — image path, name, role, badge data, description, intersection pills
- **Skills** — array of skill cards with name and tag list
- **Services** — array of service cards with number, name, desc, color variant, blob SVG
- **Pricing** — two pricing cards with tag, role, amount, desc
- **Process** — 5 steps with step number, name, desc, SVG icon (stored as JSX strings or imported)
- **Stats** — 5 stats with target number and label
- **Projects** — the full `projects` object (9 projects) with cat, title, subtitle, desc, problem, retro, tags, link, image path
- **Vision** — quote, text, values array
- **Contact** — description, email, phone, social links (with URLs), form fields config
- **Footer** — brand, copyright year, tagline

All paths reference `/assets/...` for the public directory.

### 5. Create `src/app/layout.tsx`

Root layout:
- Import Google Fonts (Poppins + Satisfy) — use `next/font` instead of CDN link for performant loading
- Import `globals.css`
- Set metadata (title, description, favicon)
- Wrap children with `<CustomCursor />` and `<Navbar />` from component imports
- Apply font classes to the `<body>` tag

### 6. Create `src/app/page.tsx`

Compose all section components in order:
```tsx
<Hero />
<About />
<Skills />
<Services />
<Pricing />
<Process />
<Stats />
<Portfolio />
<Vision />
<Contact />
<Footer />
```

### 7. Create components (one per section)

Each component:
- Is in `src/components/<Name>.tsx`
- Is a client component only if it uses state, effects, or event listeners (use `"use client"`)
- Imports data from `@/data/content`
- Uses the existing CSS class names verbatim (no CSS-in-JS, no module imports)
- Converts inline SVG from HTML to JSX (replace `class` → `className`, `style="..."` → `style={{...}}`, handle SVG-specific attributes like `stroke-width` → `strokeWidth`)

**Client components (need `"use client"`):**
- `CustomCursor.tsx` — mousemove listener, RAF loop, hover scale on links/buttons
- `Navbar.tsx` — scroll listener for `.scrolled` class toggle
- `Stats.tsx` — IntersectionObserver + animated counter
- `Portfolio.tsx` — filter state, open/close modal, display toggle
- `ProjectModal.tsx` — open/close state, keyboard listener (Escape key)
- `Contact.tsx` — form submit handler (mailto: generation)

**Server components (can be async, no `"use client"`):**
- `Hero.tsx` — static markup with orbit animations
- `About.tsx` — static markup
- `Skills.tsx` — static grid
- `Services.tsx` — static grid
- `Pricing.tsx` — static cards
- `Process.tsx` — static steps (SVG icons converted to JSX)
- `Vision.tsx` — static quote
- `Footer.tsx` — static display

### 8. Handle scroll reveal

The CSS has `.reveal` and `.visible` classes. The JS uses `IntersectionObserver` to add `.visible` on scroll. This needs to be a shared client component or hook.

**Recommendation:** Create a `ScrollReveal.tsx` client component that:
- Takes children
- Has a ref on a wrapper div with class `reveal`
- Uses `IntersectionObserver` to add `visible` class when in view

Then wrap each section's content with `<ScrollReveal>` (matching the original `reveal` class placement).

For `reveal-delay-1` through `reveal-delay-4`, pass the delay as a prop:
```tsx
<ScrollReveal delay={1}>...</ScrollReveal>
```

### 9. Handle inline SVGs (Process section)

The process section has 5 complex inline SVGs (~80 lines each). These need to be:
- Extracted as JSX strings in `content.ts`
- Rendered using `dangerouslySetInnerHTML` OR
- Better: converted to proper React SVG components in `src/components/svg/`

**Recommendation for simplicity**: Convert `class` to `className` and `style` strings to objects for each SVG, then include them directly in the component JSX. The content.ts stores a **ReactNode** (not raw string). This avoids `dangerouslySetInnerHTML`.

### 10. Update image references

Throughout `content.ts` and components, image sources like `./assets/app.webp`, `./assets/me-dans-jamais.jpeg` become `/assets/app.webp`, `/assets/me-dans-jamais.jpeg` (since `public/` is served from root).

The `about-photo-placeholder` uses `<img src="./assets/me-dans-jamais.jpeg">` → change to `<img src="/assets/me-dans-jamais.jpeg">`.

### 11. Create Dockerfile

```dockerfile
FROM node:22-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/out /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

This uses `next.config.ts` with `output: "export"` to generate a static `out/` folder, then serves it via nginx. Alternatively, if you want to run Next.js as a server (not static export), use the `standalone` output mode.

**Recommendation:** Use `output: "export"` (static HTML) since there's no dynamic backend — simpler deployment, no Node.js runtime needed on the server.

### 12. Create `.dockerignore`

```
node_modules
.git
.gitignore
.next
*.md
docs/
```

### 13. Update `next.config.ts`

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "export",
  images: {
    unoptimized: true,  // needed for static export with local images
  },
};

export default nextConfig;
```

### 14. Verify the build

```bash
npm run build
```

This generates `out/` folder with the static site. Check that all pages, assets, and routing work.

Then Docker:
```bash
docker build -t portfolio-es .
docker run -d -p 8080:80 portfolio-es
open http://localhost:8080
```

## Key Changes from Original HTML

| Original | Next.js Replacement |
|----------|-------------------|
| `index.html` (all-in-one) | Split into 16 component files + layout + data file |
| `assets/` at root | `public/assets/` |
| Google Fonts via CDN `<link>` | `next/font` for optimized loading |
| Inline `<style>` (~1100 lines) | `src/app/globals.css` |
| `#navbar` scroll class with `window.addEventListener` | React `useEffect` in `Navbar.tsx` |
| Custom cursor with `document.addEventListener` + `requestAnimationFrame` | React `useEffect` + `useRef` in `CustomCursor.tsx` |
| Scroll reveal with `IntersectionObserver` | Reusable `<ScrollReveal>` client component |
| Stat counters with `IntersectionObserver` | React `useEffect` + `useState` in `Stats.tsx` |
| Portfolio filter with `querySelectorAll` | React `useState` for active filter + `.filter()` |
| Project modal with `document.getElementById` + class toggle | React state + conditional rendering in `ProjectModal.tsx` |
| Contact form `mailto:` via `window.location.href` | React form state + `mailto:` URL generation in `Contact.tsx` |
| All data inline in HTML | Single `src/data/content.ts` with TypeScript types |

## Open Questions / Decisions for the Executor

- **ScrollReveal implementation**: Wrapping each content block individually will require careful placement of the `<ScrollReveal>` component to match the original `reveal` class positions across ~18 elements. Each must get the correct delay class.
- **SVG conversion**: The 5 process SVGs (~80 lines each) need manual conversion from HTML to JSX (`class` → `className`, `style` strings → objects, SVG attribute casing). This is mechanical but tedious — batch it carefully.
- **Font loading**: Use `next/font/google` with `Poppins` (weights 300, 400, 500, 600, 700 + italic 300, 400) and `Satisfy` (weight 400). Configure `display: "swap"` and `preload: true` for the primary weights.
